Singleton Pattern 

1.싱글톤 패턴이란?
애플리케이션 전체에서 단 하나의 인스턴스만 생성되도록 보장하는 패턴이다.
전역적으로 접근해야 하는 객체(로그, 설정, DB 커넥션 등)에 사용된다.

즉,
객체를 하나만 만든다 (unique instance)
여러 곳에서 동일한 인스턴스를 공유한다
생성자를 외부에서 호출하지 못하도록 한다

2. 왜 필요한가?
멀티스레드/대규모 시스템에서는 전역적으로 하나만 존재해야 하는 객체가 있다.

예시:
LogManager (로그 기록)
ConfigManager (설정 관리)
DB Connection Pool
Cache Manager
File Writer (파일 동시 접근 방지)

이런 객체가 여러 개 생성되면:
설정이 서로 다를 수 있고
로그 파일이 꼬일 수 있으며
DB 커넥션이 불필요하게 과다 생성될 수 있다
따라서 싱글톤은 필수적인 설계 패턴이다.

3. 구현 포인트
기능	설명
private 생성자	외부 new 방지
static 필드 _instance	단일 인스턴스 저장
static readonly _lock	멀티스레드 동기화용
lock(_lock)	인스턴스 생성 구간 보호
Instance 프로퍼티	전역 접근점 (Global Access Point)

4. 장점 (Advantages)
1) 전역 인스턴스 제공
어디서든 LogManager.Instance로 동일 객체 접근 가능.
2) 메모리 절약
여러 개 생성될 필요 없는 객체를 단 하나만 유지.
3) 데이터 일관성
설정, 로그, 캐시 등 중앙 집중식 관리 가능.
4) 멀티스레드 환경 안전(Thread-Safe)
lock을 통해 동시 접근으로 인한 중복 생성 문제를 예방.

5. 단점 (Disadvantages)
1) 결합도가 올라갈 수 있음
전역 접근 방식이기 때문에, 사용처가 많으면 테스트가 어려워질 수 있음.
2) 멀티스레드에서 성능 오버헤드
lock을 사용하면
Instance 호출 시 약간의 성능 비용이 듦
(조금 개선하려면 Double-checked locking 가능)
3) 전역 상태(Global State) 사용 → 유지보수 위험
너무 남발하면 프로그램 구조가 전역 변수처럼 복잡해짐.
